# 源码阅读助手设计说明

## 1. 产品定位

源码阅读助手是一个面向开发者的 Agent CLI，目标不是替用户写代码，而是帮助用户更快读懂陌生代码库。

它聚焦以下场景：

- 新接手项目，不知道先看什么
- 想理解某个功能的入口、调用链和模块边界
- 想快速产出源码阅读笔记、项目上手文档或模块说明
- 想把一次性的阅读过程沉淀成可复用知识

一句话定义：

> 一个专注于源码理解、阅读导航和知识沉淀的终端智能助手。

## 2. 差异化价值

相比通用 Coding Agent，这个产品的差异不在于“也能调工具”，而在于它的输出目标完全不同。

它的核心输出是：

- 阅读地图
- 推荐阅读顺序
- 模块关系说明
- 功能调用链解释
- 结构化阅读笔记

它不是以“自动改代码”为核心，而是以“帮助用户建立理解”为核心。

## 3. 目标用户

- 正在阅读开源项目的学生或开发者
- 刚加入团队、需要快速上手仓库的新成员
- 需要为项目写 onboarding 文档的人
- 想把阅读过程转成笔记或知识库的人

## 4. MVP 范围

第一阶段只做一个可用、可演示、可沉淀的 CLI，不追求完整通用 agent 平台。

### 4.1 必做能力

- 接收用户问题，例如“这个项目的入口在哪”
- 读取当前仓库路径作为工作区
- 使用搜索与读文件工具定位相关代码
- 在多轮工具调用后输出结构化答案
- 支持把结果保存成 Markdown 笔记

### 4.2 第一版工具集

- `search_files`：按文件名或路径模式搜索
- `grep_text`：按文本或符号搜索
- `read_file`：按文件读取片段
- `save_note`：保存 Markdown 阅读笔记

### 4.3 输出形态

每次回答至少组织为以下四段：

1. 问题结论
2. 关键文件
3. 阅读路径建议
4. 可选的后续问题

## 5. 非目标

以下内容明确不进入 MVP：

- 自动改代码
- 自动执行高风险 shell 命令
- 多 agent 并行调度
- MCP 插件生态
- IDE / ACP 集成
- 完整权限审批系统
- 面向所有任务的通用 agent 平台

## 6. 产品形态

产品以本地 CLI 形式运行，初始命令建议如下：

```bash
agentcli ask "这个仓库的入口在哪"
agentcli ask "认证流程相关代码怎么读"
agentcli note "请整理这个项目的模块地图"
```

默认工作目录为当前仓库根目录，也支持通过参数指定：

```bash
agentcli ask --repo /path/to/repo "这个项目的核心模块关系是什么"
```

## 7. 技术方案

### 7.1 技术栈

- Python 3.12
- Typer：CLI 入口
- Pydantic：配置与数据模型
- Rich：终端展示
- Pytest：测试

### 7.2 架构分层

- `app/cli.py`
  - CLI 命令入口
- `app/runtime.py`
  - 装配配置、工作区、系统提示词、工具注册表
- `app/agent_loop.py`
  - 管理 turn / step / tool call 回填的主循环
- `app/llm/`
  - LLM 适配层，负责模型请求和响应标准化
- `app/tools/`
  - 文件搜索、文本搜索、文件读取、笔记保存
- `app/prompts/`
  - 系统提示词模板
- `app/notes/`
  - 笔记输出格式

### 7.3 主循环思路

主循环参考 Kimi CLI 的骨架思想，但做收敛：

1. 接收用户问题
2. 生成 system prompt + user prompt
3. 调用模型
4. 若模型发起工具调用，则执行工具并回填结果
5. 重复直到模型给出最终回答
6. 可选地把回答保存为笔记

### 7.4 Runtime 装配思路

Runtime 只做四件事：

- 确定当前工作目录
- 注入系统提示词
- 注册可用工具
- 管理本次会话配置

不在 MVP 中承载复杂模式切换、session 恢复或权限桥接。

## 9. MVP 成功标准

满足以下条件即可认为第一阶段完成：

- 能在本地运行 CLI 命令
- 能对给定问题自动调用搜索和读文件工具
- 能输出结构化的源码阅读回答
- 能把结果保存为 Markdown 笔记
- 至少覆盖一条端到端测试路径

## 10. 风险与控制

### 风险 1：项目范围失控

控制方式：坚持只做“源码理解”和“笔记沉淀”，不扩展成全能 coding agent。

### 风险 2：模型回答空泛

控制方式：系统提示词强制要求引用文件路径、阅读顺序和证据来源。

### 风险 3：工具结果过多导致上下文噪音

控制方式：限制搜索结果数量、文件读取行数和单次循环步数。
